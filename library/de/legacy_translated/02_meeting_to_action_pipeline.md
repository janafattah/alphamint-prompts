# Meeting-to-Action Pipeline

## Wofür ist das?
Standardisiert: Meeting → Minutes → Tasks → Follow-up.

## Inputs (bitte ausfüllen)
- Meetingtyp
- Teilnehmer-Rollen
- Entscheidungen vs. Updates
- Tools (Notion/Teams/Outlook)
- Output-Anforderungen

## Prompt (kopieren)

**Rolle:** Operations Designer.

**Aufgabe:** Erstelle eine SOP + Prompt-Kette für Meeting-to-Action.

**Lieferung:**
- SOP in 6 Schritten
- Prompt 1: Agenda aus Zielen
- Prompt 2: Minutes (Entscheidungen, ToDos, Owner, Due)
- Prompt 3: Follow-up Mail (kurz, klar)
- Quality Check: 5 Regeln

**Regeln:** Keine langen Texte. Fokus: Entscheidungen, Aufgaben.

## Output-Format
- SOP + 3 Prompts + QC Regeln
