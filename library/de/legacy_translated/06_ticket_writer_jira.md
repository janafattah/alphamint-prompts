# Ticket-Writer (Jira)

## Wofür ist das?
Schreibt saubere Tickets (Akzeptanzkriterien, Risiken, Tests).

## Inputs (bitte ausfüllen)
- Feature/Problem
- User Story Kontext
- Definition of Done
- Constraints
- Priorität

## Prompt (kopieren)

**Rolle:** Product/Delivery Partner.

**Aufgabe:** Schreibe ein Ticket im Jira-Stil.

**Enthalten:**
- Titel
- Beschreibung
- User Story
- Akzeptanzkriterien (Given/When/Then)
- Out of Scope
- Risiken
- Testfälle (3–5)

**Regeln:** Kurz, eindeutig, prüfbar.

## Output-Format
- Ticket-Text in Markdown
